# [2026-04-19] - 测试框架规范化改造

## 变更分类

### 新增
- **规范的测试目录结构**
  - 创建 `tests/` 根目录
  - 创建 `tests/unit/` 子目录（单元测试）
  - 创建 `tests/integration/` 子目录（集成测试）
  - 创建 `tests/__init__.py`、`tests/unit/__init__.py`、`tests/integration/__init__.py` 包标识文件

- **pytest 测试框架配置**
  - 创建 `pyproject.toml` 配置文件
  - 配置 pytest 测试路径、文件命名模式、默认参数
  - 注册自定义 markers：`unit`、`integration`、`auth`、`projects`、`script_works`

- **共享 Fixtures** (`tests/conftest.py`)
  - `client`: httpx 测试客户端 fixture
  - `test_user_data`: 生成测试用户数据
  - `auth_headers`: 认证请求头
  - `test_project_data`: 测试项目数据
  - `test_script_work_data`: 测试脚本作品数据
  - `registered_user`: 自动注册测试用户
  - `logged_in_user`: 自动注册并登录
  - `test_project`: 自动创建测试项目
  - `test_script_work`: 自动创建测试脚本作品

- **单元测试文件** (`tests/unit/test_settings.py`)
  - `TestAppConfig`: APP 配置测试
  - `TestMiniMaxConfig`: MINIMAX 配置测试
  - `TestDatabaseConfig`: 数据库配置测试
  - `TestEvaluationConfig`: 评估配置测试
  - `TestDSNGeneration`: DSN 生成测试

- **集成测试文件**
  - `tests/integration/test_auth.py`: 用户认证测试（9 个测试用例）
    - `TestUserRegistration`: 用户注册测试（TC-REG-001 到 TC-REG-004）
    - `TestUserLogin`: 用户登录测试（TC-LOG-001 到 TC-LOG-005）
  - `tests/integration/test_projects.py`: 项目管理测试（7 个测试用例）
    - `TestProjectCreation`: 项目创建测试（TC-PRJ-001 到 TC-PRJ-004）
    - `TestProjectDeletion`: 项目删除测试（TC-DEL-001 到 TC-DEL-003）
  - `tests/integration/test_script_works.py`: 脚本作品测试（6 个测试用例）
    - `TestScriptWorks`: 脚本作品操作测试（TC-SCR-001 到 TC-SCR-007）

### 修改
- **依赖更新** (`requirements.txt`)
  - 添加 `pytest>=7.0.0` 依赖

- **VSCode 配置** (`.vscode/settings.json`)
  - 启用 pytest 支持：`"python.testing.pytestEnabled": true`
  - 禁用 unittest：`"python.testing.unittestEnabled": false`
  - 配置 pytest 参数：`"python.testing.pytestArgs": ["-v"]`
  - 启用保存时自动发现测试：`"python.testing.autoTestDiscoverOnSaveEnabled": true`

### 移除
- **旧测试文件**
  - 删除 `test_config.py`（原根目录下的配置测试文件）
  - 删除 `test_runner.py`（原根目录下的集成测试文件）

## 使用方式

### 命令行运行
```powershell
# 运行所有测试（使用正确的 Python 环境）
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/ -v

# 只运行单元测试
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/unit/ -v

# 按 marker 分组运行
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/ -m unit -v      # 单元测试
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/ -m auth -v      # 认证测试
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/ -m projects -v  # 项目测试
d:\python_envs\minimax_api_plan\python.exe -m pytest tests/ -m script_works -v  # 脚本测试
```

### VSCode 中运行
1. 确保选择正确的 Python 解释器：`d:\python_envs\minimax_api_plan`
2. 打开 VSCode 测试面板（烧杯图标）
3. 点击刷新按钮发现测试
4. 可点击运行单个测试或测试组

- **Python 环境**：确保使用 `d:\python_envs\minimax_api_plan` 环境，不要使用全局 Python 环境
