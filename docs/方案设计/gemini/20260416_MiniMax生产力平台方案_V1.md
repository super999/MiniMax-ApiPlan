# MiniMax AI 生产力平台设计方案 (V1)

> - **核心需求**: 基于 MiniMax 全家桶 API，打造一个支持短剧生成、多模态资产创作（图/音）、多用户隔离与权限管理的正式生产力平台。

---

## 1. 方案综述
本方案旨在将现有的 MiniMax API 演示 Demo 升级为一个具备商用潜力的“AI 创意工作站”。采用“底座 + 业务插件”的模式，核心关注点在于**数据隔离**、**角色管理**以及**跨模态（文/图/音）的生产流协同**。

## 2. 核心模块设计

### 2.1 用户与隔离系统 (IAM)
*   **多租户隔离**: 数据库层面实现 UserID 绑定，确保 A 用户的脚本和生成的敏感图片不会被 B 用户看到。
*   **角色控制 (RBAC)**:
    *   **普通用户**: 只能访问自己的项目空间，有 Token 使用限额。
    *   **管理员**: 拥有全局看板，可配置系统 API Key，管理用户额度，查看全站消耗报表。

### 2.2 项目空间 (Project-Based Workspace)
*   引入“项目”概念，将零散的 API 调用整合为“任务流”。
*   一个短剧项目可包含：剧本、分镜草图、角色预设音频、参考文档等。

### 2.3 业务引擎 (Productivity Engines)

#### **创意写作引擎 (Creative Writing)**
*   **场景化生成**: 预设“短剧脚本”、“网文大纲”、“小红书文案”等专用 Prompt。
*   **长文本一致性**: 利用 MiniMax 的上下文管理，保持长篇剧情逻辑不崩。

#### **视觉资产引擎 (Visual Studio)**
*   **多模式绘图**: 集成 T2I (文生图) 和 I2I (图生图)。
*   **角色锁定**: 提供“以图生图”或 Prompt 参数优化，尽量维持同一项目中角色形象的连贯性。

#### **音频实验室 (Audio Lab)**
*   **角色配音编辑器**: 为剧本中的不同角色分配 MiniMax 复刻音色。
*   **一键合成**: 将整段剧本脚本一键转为多角色对白音频包。

## 3. 技术架构建议 (Scalability)
*   **API 抽象层**: 将 MiniMax 各类接口（T2T, T2I, T2A）进行 Service 级封装，方便未来引入视频生成等新能力。
*   **异步任务管理**: 针对绘图等高耗时请求，采用 Task Queue 机制，前端通过 WebSocket 或轮询获取生成进度。
*   **持久化层**: 建议引入数据库（如 SQLite 或 PostgreSQL）存储用户记录、项目关系和文件索引。

## 4. 改进建议与风险评价
1.  **并发问题**: 生产力平台可能有多人同时绘图，需考虑 API 的 Rate Limit。
2.  **安全管控**: 鉴于涉及图文生成，建议在后端加入敏感词过滤逻辑。
3.  **成本监控**: 定时任务同步 MiniMax 余额，防止 Token 爆表。
